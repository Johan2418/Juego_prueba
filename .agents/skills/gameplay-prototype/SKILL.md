# Gameplay Prototype Skill

Use this skill to build small playable features.

## Rules

- Implement one mechanic at a time.
- Keep the result testable.
- Do not overengineer.
- Avoid adding large systems before the prototype works.
- Explain exactly how to test the mechanic in Unity.
- If a placeholder is needed, use a simple and replaceable solution.

## Prototype priority

1. Movement.
2. Camera.
3. Collisions.
4. One core mechanic.
5. One small level.
6. Win/end condition.
7. UI feedback.
