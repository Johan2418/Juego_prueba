# Game Design Docs Skill

Use this skill to keep documentation useful and short.

## Rules

- Keep documentation aligned with the current demo.
- Do not write huge design documents too early.
- Update docs when mechanics, controls or structure change.
- Prefer checklists and short explanations.
- Mark pending decisions clearly.

## Documents

- README.md: how to open and understand the project.
- GAME_DESIGN.md: what the game is.
- ROADMAP.md: what will be built next.
- PLANS.md: how Codex should plan larger tasks.
- CODE_REVIEW.md: how to review changes.
