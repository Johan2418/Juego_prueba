# Game Design Document - Demo Inicial

## Concepto

Videojuego 2D con estilo pixel art.

## Tipo de camara

Elegir una opcion:

- [ ] Lateral tipo metroidvania/plataformas
- [ ] Vista superior tipo aventura/Zelda clasico

## Objetivo de la demo

El jugador debe poder completar una pequena seccion jugable de 3 a 5 minutos.

## Mecanicas principales

Pendiente de definir:

- Movimiento
- Interaccion
- Combate o evasion
- Exploracion
- Objetivo final del nivel

## Controles iniciales

### PC

- Movimiento: WASD o flechas
- Accion/interactuar: E
- Ataque: clic izquierdo o J
- Pausa: Escape

### Movil, si aplica despues

- Joystick virtual
- Boton de accion
- Boton de ataque

## Personaje principal

Pendiente de definir:
- Nombre
- Motivacion
- Apariencia
- Animaciones necesarias

## Mundo o escenario

Pendiente de definir:
- Tema visual
- Ambientacion
- Obstaculos
- Enemigos
- NPCs

## Loop jugable inicial

Ejemplo:
1. El jugador aparece en el nivel.
2. Aprende a moverse.
3. Encuentra un obstaculo o enemigo.
4. Usa la mecanica principal.
5. Avanza a una zona final.
6. Completa la demo.

## Alcance de la primera demo

Debe incluir:
- Movimiento funcional.
- Camara funcional.
- Un nivel pequeno.
- Una mecanica principal.
- Una condicion de victoria simple.
- UI minima.
- Audio basico opcional.

No debe incluir todavia:
- Sistema grande de inventario.
- Arbol de habilidades.
- Guardado complejo.
- Tienda.
- Multiples mundos.
- Multijugador.
